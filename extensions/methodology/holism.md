# 整体论 / 整体观

## 这是什么

不是"从大局出发"的模糊态度。整体论是一个具体的操作：**在深入任何局部细节之前，先画出系统的全局结构图，标出你当前站在哪一层、上游是谁、下游是谁、同层是谁。** 深入局部时不断回到这张图上确认位置。分析完成后用这张图验证局部结论在每一层都说得通。

> 整体观的本质不是"看全局"——是"每一步都知道自己站在全局的哪个位置"。你不知道自己站在哪，深入局部就会迷路。

## 四步操作流程

### Step 1: 画全局结构图

画出从入口到出口的完整调用链/数据流。不追求完备——追求"当前问题可能涉及的每一层都在图上了"。

```
[整体观·Step1] 全局结构：
输入：______
  → 层 1：______（职责：______）
  → 层 2：______（职责：______）
  → 层 3：______（职责：______）
  → 层 N：______（职责：______）
输出：______

当前问题现象出现在：层 N（输出层）
可能的病根分布在：层 1 ~ 层 N-1（上游任意一层出错都可能导致输出异常）
```

> 涉及 3+ 层/组件时，必须画 Mermaid 图。文字描述层级关系不如一张图。

### Step 2: 标注当前位置

说出你当前站在哪一层，正在分析什么，上游和下游分别是什么。

```
[整体观·Step2] 当前位置：
我们站在 ______（具体函数/模块/层）。
上游：______（谁调用我们，传递了什么输入）
下游：______（我们调用谁，期望什么输出）
同层：______（和我们在同一层的有什么）
```

### Step 3: 标认知边界

在图上的每个节点标出"已知"和"未知"。哪些部分是我们确定的，哪些还需要验证？

```
[整体观·Step3] 认知边界：
层 1：____ [已知] / [部分已知：____] / [未知]
层 2：____ [已知] / [部分已知：____] / [未知]
层 3：____ [已知] / [部分已知：____] / [未知]
```

### Step 4: 局部深入 + 全局回望

每次深入一个局部之后，回到全局图上：这个局部的结论在全局每一层都成立吗？

```
[整体观·Step4] 全局回望：
当前局部分析的结论：______
这个结论在层 1 说得通吗？（上游——这个结论和上游的行为一致吗？）
这个结论在层 N-1 说得通吗？（下游——下游的表现和这个结论一致吗？）
这个结论在全局说得通吗？（有没有任何一层需要"特殊解释"？）
```

## 四种假执行模式

### 假模式 1：画了图但图里没有自己

```
❌ 画了一个漂亮的 Mermaid 架构图，但接下来的分析完全脱离了图。
   图只是装饰——你还是在逐层深入时丢失了全局感。

✅ 每次进入新的局部时，回到图上："好，现在我们从 ext4_read_file
   往下进入 bread 层。在全局图上，这是从第 4 层进入第 5 层。"
```

### 假模式 2：图太抽象——"应用层→内核层→硬件层"

```
❌ 三层抽象等于没画。"内核层"包含了 50 个可能出错的模块。

✅ 图的粒度必须对应当前问题。如果排查的是 I/O 问题，
   画到 I/O 栈的具体函数级：exec → loadseg → readi → ext4_read_file → bread → bget → virtio_disk_rw。
   如果排查的是调度问题，画到调度路径的具体函数级。
```

### 假模式 3：认知边界全标"已知"

```
❌ 每个节点都标 [已知]——要么你真的全知道（不可能，在 debug 场景），
   要么你在假装知道。

✅ 至少有 30% 的节点标注"部分已知"或"未知"。
   认知边界的价值不在于已知区域，在于未知区域——那是下一步要探索的方向。
```

### 假模式 4：只管画不管交叉验证

```
❌ 拆解完后逐层验证了，但没做全局回望。"每层都对，所以整体对"——
   这只是必要条件，不充分。局部都对的部件拼起来可能因为接口不匹配而出错。

✅ Step4 的全局回望不可跳过。"exec 层认为 ELF 的 offset 是文件偏移，
   ext4 层认为 offset 是逻辑块偏移——虽然每层在自己的假设下都正确，
   但接口契约不一致导致了整体错误。"
```

## 不同领域的示例

### 示例 1: 后端 — 支付回调处理失败

```
[整体观·Step1]
用户支付 → 支付宝/微信 → callback URL → Nginx → App Server → Order Service → Payment Service → DB
                                                                          → 消息队列 → 库存服务

[整体观·Step2]
当前位置：我们在 Payment Service 层。
上游：Nginx → App Server → Order Service —— 都在正常转发请求
下游：DB（写支付状态）+ 消息队列（通知库存）
同层：Order Service（订单状态管理）

[整体观·Step3]
Nginx: [已知] access log 收到了回调请求，状态码 200
App Server: [已知] 收到了请求，转发到 Order Service
Order Service: [部分已知] 调用了 Payment Service，但我们没确认参数是否正确
Payment Service: [未知] —— 当前排查目标
DB: [已知] payments 表没有新记录
消息队列: [未知]

[整体观·Step4]
结论：Payment Service 的签名验证失败，因为密钥配置错误。
回望：Nginx 返回 200（Nginx 只是反向代理，不处理签名）✓
     App Server 正常转发 ✓
     Order Service 正常调用 Payment Service ✓
     Payment Service 签名失败返回错误 → DB 无记录 ✓
     → 全局一致：每一层的行为都吻合。
```

### 示例 2: 编译器 — 生成的代码不正确

```
[整体观·Step1]
源码 → Lexer → Parser → AST → Semantic Analyzer → IR → Optimizer → CodeGen → 汇编

[整体观·Step2]
当前站在 CodeGen 层。上游是 IR（经过优化的中间表示），下游是汇编器。
现象是生成的汇编指令错误，但病根可能在上游任何一层——IR 错了，CodeGen 再正确也没用。

[整体观·Step3]
Lexer → AST: [已知] 解析无报错
Semantic Analyzer: [部分已知] 类型检查通过，但可能漏了隐式转换
IR: [未知] —— 需要 dump IR 对比
Optimizer: [未知] —— 可能是错误的优化 pass

[整体观·Step4]
结论：Optimizer 的一个 pass 错误地移动了 load 指令越过 store 指令。
回望：Lexer→AST→Semantic 正确（源码的语义没被破坏）✓
     IR 在优化前正确 ✓
     Optimizer 破坏了指令依赖 → CodeGen 生成了错误代码 ✓
     → 全局一致，根因在 Optimizer。
```

## 深度自检

| 自测问题 | 及格线 | 不及格特征 |
|---------|-------|-----------|
| 我画的图能不能让一个不了解这个问题的人看出调用/数据流的方向？ | 箭头有方向，每层有职责 | 只有框没有边，或边没有标注 |
| 我是否在每次深入局部前都回到图上确认了位置？ | 每次深入前有"我们现在进入层 X" | 图只在开头画了一次，后面再也没用 |
| 认知边界是否有 30%+ 的"部分已知"/"未知"？ | 有诚实的未知标注 | 全标已知——你实际上是在猜 |

## 转换信号

### 何时进入
- 任何对话开始 → 第一步永远是整体观定位
- 涉及 3+ 组件/模块 → 必须画图
- 排查卡住了 → 可能是你在某一层钻太深丢了全局，回到整体观
- [还原论] 拆解完成后 → 整体观复原

### 何时退出
- 全局图清晰 + 当前位置明确 + 认知边界标好 → 可以进入 [现象学] 或 [还原论]
- 发现某个"未知"节点是关键 → 进入 [格物致知] 读源码/文档
- 全局回望确认所有层自洽 → 转 [辩证法] 回看

## 配合使用的工具

| 在前 | 本工具 | 在后 |
|------|--------|------|
| （无——整体观是第一步） | **整体观** | [现象学] 收集事实 / [还原论] 拆解 |
| [还原论] 拆解完成后 → | | [收束] 划界 / [辩证法] 回看 |
| | | [结构主义] 补充关系网络的细节 |

## 标注前自检清单

- [ ] 是否画了全局结构图？如果 3+ 层/组件，是否用了 Mermaid？
- [ ] 是否明确标注了"我们现在站在哪一层"？
- [ ] 是否标注了已知和未知区域（认知边界）？
- [ ] 深入局部后是否回到图上做了全局回望？
- [ ] 图的粒度是否匹配当前问题（不能太粗——"内核层"；不能太细——每个变量都画）？

**不画图就深入局部 = 你不是师兄。**
