# 整体论 / 整体观

## 触发条件

- 任何对话开始时——先定位再深入
- 涉及 3+ 组件/模块的讨论
- 刚完成还原论拆解，需要装回去
- 解释一个局部改动对全局的影响

## 关键问句

- 这个问题处在系统的哪一层？上游是什么？下游是什么？
- 我现在看到的是现象、局部机制、还是整体结构？
- 这个局部改动对整体有什么影响？
- 装回整体后，行为是否在每个层次都解释得通？

## 使用方式

在对话中以 `[整体观]` 标注。两种使用模式：① 启动时——先画全局架构图（Mermaid），标注当前位置；② 收束时——确认局部结论在全局各层都成立。

## 示例

> `[整体观]` 我们先定位一下。当前问题出在 `bread()` / `bget()` 这一层——它在整个 I/O 栈的位置是：
> exec → loadseg → readi → ext4_read_file → **bread → bget** → virtio_disk_rw
> 上游是文件系统层（ext4_read_file），下游是块设备驱动（virtio_disk_rw）。现在问题是：上游给对了块号，下游也能读盘，但中间返回了错误数据。

> ⚠️ 整体观不是"看一遍架构图"，而是每一步都知道自己站在哪一层。画完图后，每次深入局部前都回到图上确认位置。
